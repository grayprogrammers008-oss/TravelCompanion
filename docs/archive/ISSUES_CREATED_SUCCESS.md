## 💡 Collaboration Tips

### Daily Workflow

1. **Morning**: Pull latest changes
   ```bash
   git pull origin main
   ```

2. **Create Feature Branch**:
   ```bash
   git checkout -b feature/issue-1-edit-trip
   ```

3. **Work on Issue**: Make changes, commit frequently
   ```bash
   git commit -m "feat: add trip edit form (#1)"
   ```

4. **Push Branch**:
   ```bash
   git push origin feature/issue-1-edit-trip
   ```

5. **Create Pull Request**: Request review from team member

6. **Update Issue**: Comment with progress, check off completed tasks

---

### Branch Naming Convention

```
feature/issue-1-edit-trip         # Features
bugfix/issue-18-overflow-error    # Bug fixes
docs/issue-20-user-guide          # Documentation
```

---

### Commit Message Format

```
feat: add trip edit functionality (#1)
fix: resolve Stack overflow in trip cards (#18)
docs: add user guide for expenses (#20)
test: add unit tests for trip repository (#13)
```

---

### Closing Issues

When an issue is complete, use this in your commit message:
```bash
git commit -m "feat: complete trip edit functionality

Closes #1"
```

Or add "Closes #X" in the Pull Request description.
